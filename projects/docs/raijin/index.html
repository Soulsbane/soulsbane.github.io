    API documentation

    Module Description
    raijin.patterns.signals A simple implementation of the signals pattern.
    raijin.types.boolean Contains a boolean type which can convert boolean like values(numeric, string, etc) to a bool.
    raijin.types.callbacks Basic interface for working with callbacks(wrapped functions/delegates).
    raijin.types.dynamic Contains a simple struct(DynamicType) that can store strings, integers, booleans and decimals only.
    raijin.types.functions Various functions to determine types at runtime.
    raijin.types.queue Simple module for dealing with queues.
    raijin.types.records A simple module for working with a record format.
    raijin.types.stack Simple module for dealing with stacks.
    raijin.types.structoptions A key value configuration format using compile time reflection and structs.
    raijin.utils.array Various functions for working with arrays.
    raijin.utils.debugtools Functions for outputing a debug string.
    raijin.utils.file Various functions for dealing with files.
    raijin.utils.getoptmixin Enables the autogeneration of getopt commandline parameters based on a passed structs members.
    raijin.utils.path Various functions for dealing with path based manipulation and retrieval.
    raijin.utils.process Small wrapper functions for std.process functions.
    raijin.utils.string This module contains a few additons to handling strings.
    raijin.appconfig This module combines the functionality of KeyValueConfig and ConfigPath into one class.
    raijin.cmdline A simple module that handles user input on the command line.
    raijin.commander Allows command line arguments to be mapped to a function using the types passed using the Commander mixin.
    raijin.commandlineargs This module contains a simple class that processes command line arguments and stores them in a hashtable. The hash table is static so this class object can be created multiple times without having to reinitialize it when the arguments from main.
    raijin.configpath Provides an easy way to work the users configuration directory.
    raijin.keyvalueconfig This module manages a config file format in the form of key=value. Much like an ini file but simpler.
    raijin.timers Add support for creating a repeating timer.
